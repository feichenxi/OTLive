# OTLive 规模化直播矩阵系统 - 项目功能与说明

## 一、项目概述

OTLive 是一个基于Windows和ESP32-S3的规模化直播矩阵系统，用于管理多个直播间的设备控制和音频对讲功能。

## 二、系统组成

### 2.1 硬件组成

- **控制中心**：Windows PC
- **终端设备**：多个ESP32-S3 N16R8开发板（每个房间一个）
- **音频模块**：PCM5102MK2.0 I2S功放模块 + USB麦克风
- **通信方式**：WiFi（TCP/UDP/WebSocket）

### 2.2 软件组成

- **控制中心**：Python 3.x + Flask + WebSocket
- **ESP32-S3固件**：Arduino框架 + PlatformIO
- **数据库**：SQLite（本地配置） + MySQL（远程消息）
- **AI功能**：FallCatLive模块（AI文本生成 + 语音合成）

## 三、核心功能

### 3.1 设备控制

- **触发装置**：控制直播间触发装置开关
- **常动装置**：控制直播间常动装置开关
- **可编程设备**：支持10个可编程GPIO设备控制
- **实时状态**：WebSocket实时推送设备状态

### 3.2 音频功能

- **房间对讲**：UDP实时音频传输
- **WAV文件播放**：支持HTTP URL下载和流式播放
- **噪声门**：可配置的音频噪声门功能
- **音量控制**：房间级别的音量调节

### 3.3 礼物触发

- **礼物监听**：从MySQL数据库监听礼物消息
- **触发配置**：每个房间可配置不同礼物的触发规则
- **触发队列**：智能触发队列管理
- **日志记录**：完整的触发日志

### 3.4 AI功能

- **文本生成**：根据礼物信息生成个性化感谢语
- **语音合成**：将感谢语转换为语音
- **房间专属提示词**：每个直播间可配置专属AI提示词

### 3.5 授权管理

- **机器码授权**：基于机器码的授权验证
- **授权类型**：试用/月付/年付/永久
- **房间限制**：可配置最大房间数
- **周期性验证**：后台自动验证授权状态

### 4.2 ESP32-S3固件功能

- WiFi连接管理（自动重连）
- Web服务器（HTTP API）
- TCP服务器（设备控制，短连接非阻塞）
- UDP服务器（音频接收）
- UDP心跳广播（ESP32主动向PC发送心跳）
- I2S音频输出（PCM5102，FreeRTOS独立任务）
- GPIO设备控制
- WiFi断线自动重连

## 五、网络通信协议

### 5.1 Windows → ESP32-S3通信

- **TCP控制命令**：通过TCP短连接发送设备控制命令（非阻塞，单命令单连接）
- **HTTP API**：通过HTTP接口控制ESP32-S3设备
- **UDP音频流**：通过UDP发送音频数据

### 5.2 ESP32-S3 → Windows通信

- **UDP心跳广播**：ESP32每5秒主动向PC发送UDP心跳包，包含设备状态、RSSI、音频状态等信息
- **心跳内容**：JSON格式，包含type、ip、rssi、audio_active、intercom_enabled、uptime、free_heap、devices等字段
- **心跳端口**：UDP 9999（PC端监听）

### 5.3 心跳机制（v4.52+）

心跳方向已从"PC轮询ESP32"改为"ESP32主动推送"：

| 项目 | 旧方案 | 新方案 |
|------|--------|--------|
| 方向 | PC → ESP32 | ESP32 → PC |
| 协议 | TCP长连接 | UDP广播 |
| 间隔 | 10-30秒 | 5秒 |
| 阻塞 | 串行轮询，WAV播放时阻塞 | 完全非阻塞 |
| 离线检测 | TCP连接失败 | 心跳超时（15秒） |
| 设备状态 | PC主动查询 | 心跳包携带 |

### 5.4 WebSocket实时通信

- `room_status_update`：房间状态更新
- `device_state_change`：设备状态变化
- `audio_status_update`：音频状态更新

## 六、配置文件

### 6.1 config.ini

主要配置项：
- `[system]`：系统名称、版本、调试模式
- `[server]`：Web服务器监听地址和端口
- `[audio]`：音频采样率、通道数、缓冲区大小
- `[network]`：UDP/TCP/WebSocket端口
- `[logging]`：日志级别、文件大小、备份数
- `[performance]`：设备响应超时、心跳间隔、心跳端口、心跳超时、房间监控开关
- `[gift_trigger]`：礼物触发服务配置

### 6.2 config.db

SQLite本地数据库，存储：
- 房间配置（rooms表）
- 设备配置（devices表）
- 礼物配置（gifts表）
- 礼物触发配置（gift_triggers表）
- 系统配置（system_config表）

## 七、系统要求

### 7.1 Windows控制中心

- 操作系统：Windows 10/11
- Python版本：3.8+
- 内存：4GB+
- 网络：WiFi或以太网

### 7.2 ESP32-S3终端

- 芯片：ESP32-S3 N16R8
- 固件：Arduino框架
- 开发工具：PlatformIO

## 八、快速开始

### 8.1 控制中心启动

```bash
cd iot
python main.py
```

### 8.2 访问Web界面

启动后在浏览器中打开：http://localhost:5000

### 8.3 ESP32-S3固件上传

使用PlatformIO或esptool上传固件到ESP32-S3开发板。
